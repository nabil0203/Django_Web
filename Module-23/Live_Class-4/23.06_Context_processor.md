# Context Processor
- Inside Project -> [Click Here](https://github.com/nabil0203/ShopNest/blob/main/shop/context_processor.py)
----------

- Till now we used `context` in specific view
- Only specific view can access the data of that context
-------
- But if any data is required globally, then what to do?
    - use **Context Processor**
- A context processor is a Python function that injects variables/data into the context of every Django template globally.
- Means it is global for every view


--------
--------

## Context vs Context Processor

### Normal context
- Passed from a specific view
- Only that one template/view can access it

### Context Processor
- Global data can be used everywhere
